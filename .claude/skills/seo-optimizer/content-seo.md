## E-E-A-T & Search Intent

### Four intent types — identify before writing

| Intent | Signal phrases | Content goal |
|---|---|---|
| Informational | "what is", "how to", "why does" | Comprehensive answer, structured sections |
| Navigational | brand name, product name | Fast, clear destination |
| Commercial | "best", "vs", "review", "alternatives" | Comparison, evidence, social proof |
| Transactional | "buy", "price", "discount", "sign up" | Frictionless CTA, trust signals above fold |

### Match intent in the first 100 words

- State the answer or outcome **before** explaining it (BLUF).
- For informational queries: give the definition or key finding in sentence 1.
- For commercial queries: state the winner or top recommendation in the opening.
- For transactional: lead with the value proposition and primary CTA.
- 97% of AI Overview citations come from pages already in the top 20 organic results — rank first by matching intent perfectly.

### Topical authority signals

- Cover the full semantic cluster around a topic, not just the head keyword.
- Link to supporting articles that cover subtopics in depth.
- Reference primary sources: studies, official docs, original research.
- Add author credentials (name, title, relevant expertise) near the byline.
- Include a last-updated date on time-sensitive content.

## BLUF — Answer First, Explain Second

Bottom Line Up Front (BLUF) is the single highest-impact writing pattern for AI Overviews and featured snippets.

**Structure every major section as:**

```
[Direct answer in 1–2 sentences]
[Supporting evidence, context, or steps]
[Optional: nuance, caveats, examples]
```

**Example — bad (buries the lead):**
> Search engine optimization has been evolving rapidly. Many factors influence rankings.
> Google uses hundreds of signals. One of the most important things you can do is...

**Example — good (BLUF):**
> Core Web Vitals directly impact Google rankings. Sites that move from "Poor" to
> "Good" INP scores report a 25% increase in conversion rates. To improve your score:
> 1. Reduce JavaScript execution time on the main thread.
> 2. Use island architecture (partial hydration) for interactive components.

## Elite Copywriting Rules

### Active voice — always

- **Passive:** "The feature was released by the team."
- **Active:** "The team released the feature."

Exception: passive is acceptable in technical documentation where the actor is unknown or irrelevant.

### Sentence length — vary deliberately

- Average sentence: 15–20 words.
- Use short sentences (≤8 words) for impact after a complex idea.
- Never write three consecutive sentences of the same length.
- Maximum paragraph length: 3 sentences for body copy, 5 for technical explanation.

### Bucket brigades — keep readers moving

Insert transitional "bridge" phrases at the start of new paragraphs to pull readers forward:

- "Here's the problem:"
- "But there's a catch."
- "The data tells a different story."
- "This is where most teams go wrong:"
- "What does that mean in practice?"

### Hooks — first sentence must earn the second

Lead with one of:
- A surprising statistic: "Only 47% of websites pass all Core Web Vitals thresholds."
- A direct contradiction: "More content doesn't mean better rankings."
- A provocative question: "What if your title tag is costing you 34% of your clicks?"
- The most important outcome: "Fix these three meta tags and double your CTR."

### Readability standards

- **Flesch Reading Ease:** Target 60–70 (plain English) for B2B, 70–80 for B2C.
- **Headings:** Every 200–300 words of body copy must have an H2 or H3.
- **Bullets:** Use for lists of 3+ parallel items. Never write a bullet longer than 2 lines.
- **Bold:** Highlight 1–2 key phrases per section. Never bold entire sentences.
- **White space:** Single blank line between paragraphs. No walls of text.

## Banned AI-Speak Word List

Remove every instance of these words and phrases from any content being optimized.
Replace with direct, specific language.

| Banned term | Replace with |
|---|---|
| delve (into) | explore, examine, cover, look at |
| tapestry | mix, range, combination |
| testament | proof, sign, evidence |
| nuanced | specific, detailed, precise |
| landscape | field, industry, market, space |
| leverage (as a verb) | use, apply, deploy |
| utilize | use |
| in today's fast-paced world | [delete entirely] |
| it's important to note that | [delete entirely] |
| in conclusion | [delete — just write the conclusion] |
| seamless | smooth, frictionless, integrated |
| robust | strong, reliable, powerful |
| cutting-edge | [replace with the specific feature] |
| innovative | [replace with what it actually does] |
| game-changer | [replace with the concrete benefit] |
| comprehensive solution | [state what it actually solves] |
| I'll help you with that | [delete — just help] |
| Certainly! / Absolutely! / Of course! | [delete entirely] |
| As an AI language model | [never write this] |

**Rule:** Every edit that removes AI-speak must add a specific, concrete replacement. Vagueness is the enemy of ranking.

## Keyword Integration

### Primary keyword placement rules

| Location | Rule |
|---|---|
| H1 | Primary keyword in the first 5 words |
| First 100 words | Primary keyword appears at least once, naturally |
| At least one H2 | Contains primary keyword or close variant |
| Last paragraph | Primary keyword or core concept reinforced |
| Title tag | Primary keyword in position 1 |
| Meta description | Primary keyword within first 20 words |
| Image alt text | Descriptive alt including keyword where natural |
| URL slug | Primary keyword, hyphens only, no stop words |

### LSI / semantic keyword integration

- Identify 5–10 semantically related terms using the primary keyword's search cluster.
- Distribute naturally across H2s, H3s, and body copy.
- Never force: if a semantic keyword doesn't fit the sentence, skip it.
- Use definition lists (`<dl>`) and tables for structured facts — these are heavily weighted by AI Overview extraction.

### Keyword density

- **Primary keyword:** 1–2% of total word count (once per ~75 words).
- **LSI keywords:** Each 0.5–1% (once per ~100–150 words each).
- **Never repeat the exact phrase more than 3x in consecutive paragraphs.**

### URL slug rules

- All lowercase, hyphens as separators, no underscores.
- Include primary keyword.
- Remove stop words: a, an, the, and, or, but, in, on, at.
- Maximum 5–7 words.
- **Bad:** `/blog/The-Best-Ways-To-Optimize-Your-Website-For-SEO-In-2026`
- **Good:** `/blog/website-seo-optimization-2026`

## Click-Through Optimization (Title & Meta)

### Title tag rules

- **Maximum 60 characters** (Google truncates at ~600px, ~60 chars).
- Include primary keyword in the first 3–5 words.
- Use a power word or number to increase CTR: "How to", "X Ways", "The [Year] Guide".
- Separate the brand with ` | ` or ` – ` at the end if space allows.
- **Never duplicate** title tags across pages.

**Title tag formula options:**

```
[Primary Keyword]: [Benefit/Promise] | [Brand]
[Number] [Primary Keyword] [Action] in [Year] | [Brand]
How to [Achieve Outcome] with [Primary Keyword] | [Brand]
```

**Examples:**
- `Core Web Vitals: Fix INP Scores Fast | DebugBear` (52 chars)
- `10 SEO Best Practices for 2026 | Search Engine Journal` (54 chars)
- `Next.js SEO: Metadata, Schema & Speed Guide | Vercel` (53 chars)

### Meta description rules

- **Maximum 155 characters** (Google truncates beyond ~920px).
- Include primary keyword within first 20 words.
- Include a clear call-to-action verb: Learn, Discover, Get, Download, Start.
- Mention a specific benefit or outcome.
- **Never duplicate** meta descriptions across pages.
- Write as a complete sentence with terminal punctuation.

**Meta description formula:**

```
[Primary keyword statement]. [Specific benefit or outcome]. [CTA verb] + [next step].
```

**Example:**
> Core Web Vitals failures cost rankings and conversions. Learn how to fix LCP, INP, and CLS
> with our 2026 step-by-step audit guide. Start optimizing today.
> (155 chars exactly)

## Content Structure Standards

### Page structure hierarchy

```
H1 (1× per page) — Primary keyword, matches search intent
  H2 — Major section, keyword variant or LSI term
    H3 — Subsection, supporting concept
    H3 — Subsection
  H2 — Next major section
    H3 — Subsection
```

**Rules:**
- One H1 per page — never more.
- H2s and H3s must follow logical hierarchy (no skipping levels).
- H2/H3 text must provide standalone context (a reader skimming headings must understand the page).

### FAQ blocks

Add a FAQ section to every informational page:

- 4–8 questions mirroring real "People Also Ask" phrasing.
- Each answer: 2–4 sentences, BLUF format.
- Wrap in `<details>`/`<summary>` or semantic HTML for scannability.
- These blocks are high-priority targets for AI Overview extraction.

### Content length benchmarks

| Page type | Target word count |
|---|---|
| Blog post / article | 1,200–2,500 words |
| Product page | 500–900 words |
| Landing page (commercial) | 800–1,500 words |
| FAQ page | 1,000–2,000 words |
| Category page | 400–700 words |

Quality over length — never pad. Every sentence must add information or advance intent.

## E-E-A-T Signal Checklist

Apply to every content piece:

- [ ] Author name visible, linked to an author bio page
- [ ] Author bio includes: credentials, years of experience, relevant publications/links
- [ ] `author` schema markup present (see `schema-templates.json`)
- [ ] Publication date and last-updated date visible
- [ ] Primary claims cite a source (link to study, official doc, or data)
- [ ] No unsubstantiated superlatives ("the best", "the only", "guaranteed")
- [ ] External links to authoritative sources (gov, edu, major publications)
- [ ] Internal links to supporting content that covers subtopics
- [ ] Social proof present where relevant (review count, user number, case study)
- [ ] Content is original — no copy-paste from other pages on the site
