# GitHub Repository Settings Recommendations

Since you are preparing to make this repository public on a free, personal GitHub account, here are the latest recommended security settings and branch protection rules for 2024. 

## 1. Wiping PR History (Required for a Clean Start)
Currently, GitHub stores all old Pull Requests (like #107) and issues in its backend database. Even though we scrubbed the Git history locally, the old PRs on GitHub will still reference old commits that could contain sensitive information. **GitHub does not allow you to delete PR history from an existing repository.**

**How to get a fresh start (PRs start at #1, no sensitive history):**
1. Go to your current repository on GitHub (`savethepolarbears/agenthaus-marketplace`), navigate to **Settings**, scroll to the bottom (Danger Zone), and click **Delete this repository**. 
2. Create a **new** repository on GitHub with the exact same name (`agenthaus-marketplace`). Do not initialize it with a README or license.
3. On your local machine, run the following to push this clean history to the new repository:
   ```bash
   git remote remove origin
   git remote add origin https://github.com/savethepolarbears/agenthaus-marketplace.git
   git push -u origin main
   ```

## 2. Branch Protection Rules
Once your repository is public, you can enable Branch Protection rules (a feature previously only for Pro accounts, but now available for public repos on Free accounts).

1. Go to **Settings** > **Branches**.
2. Click **Add branch ruleset**.
3. Name it `Protect Main` and set the **Target branches** to `Default branch` (`main`).
4. Enable the following protections:
   - **Require a pull request before merging:** Check this. This prevents anyone (including you) from pushing code directly to `main` without a PR. 
     - *Require approvals:* You can set this to `1` or disable it if you are the only contributor and want to merge your own PRs without waiting for someone else.
   - **Require status checks to pass:** Check this if you set up GitHub Actions (e.g., for linting or testing).
   - **Require linear history:** Check this. It prevents messy merge commits and keeps your git history purely rebased.
   - **Do not allow bypassing the above settings:** Check this to ensure even administrators follow the rules.

## 3. General Security Features
GitHub provides several free security tools for public repositories to prevent credential leaks and vulnerabilities:

1. Go to **Settings** > **Code security and analysis**.
2. **Dependabot alerts:** Enable this to get notified if your dependencies have known vulnerabilities.
3. **Secret scanning:** Enable this. It will scan your codebase for accidentally committed API keys and tokens.
4. **Push protection:** Enable this. It will automatically block commits that contain identifiable secrets (like OpenAI or Stripe keys) before they even reach GitHub.
5. **Private vulnerability reporting:** Enable this feature. It allows security researchers to privately report vulnerabilities to you (via the `SECURITY.md` file we just created) without exposing the issue to the public via the issue tracker.

## 4. Account Security
Ensure your personal GitHub account has **Two-Factor Authentication (2FA)** enabled, as compromised maintainer accounts are the leading cause of supply chain attacks in open-source projects.
